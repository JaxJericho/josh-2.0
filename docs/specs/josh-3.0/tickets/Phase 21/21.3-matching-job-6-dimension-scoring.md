# Ticket 21.3 — Matching Job: 6-Dimension Scoring (M)

## Role

Principal Systems Architect & Full-Stack Engineer (algorithm correctness, legacy removal)

## Phase

Phase 21 — Matching System Updates, Destructive Migrations, And Legacy Removal

## Complexity

Medium

## What This Ticket Builds

Updates matching candidate scoring to use the **3.0 six-dimension coordination profile model** and 3.0 weights. Removes remaining 2.0 fingerprint factor references from scoring logic.

## Scope Locks (Anti-Scope-Creep)

### Allowed Change Surface

- Scoring code in `packages/core/` used by matching
- Matching job call sites that invoke scoring (`scripts/run-matching-job.mjs`)
- Tests covering scoring behavior

### Forbidden

- No schema changes (no migrations)
- No changes to candidate selection logic (except wiring to new scoring)
- No changes to eligibility rules
- No new scoring models, no new dimensions, no new weights beyond the 3.0 spec

### No New Concepts Rule

Do not introduce new scoring frameworks/abstractions. Update existing scoring pipeline.

### Production Ready Definition

- Scoring uses only 3.0 six-dimension inputs
- Weights match `Compatibility Scoring And Matching Algorithm 3.0`
- Incomplete profiles are handled deterministically (reduced confidence, not excluded)
- Tests validate relative scoring
- `pnpm lint`, `pnpm typecheck`, `pnpm build` pass; `pnpm test` passes except known baseline guardrail (if still present)

## Requirements (Exact)

- Read `CoordinationDimensions` from the 3.0 column (`coordination_dimensions`) via the existing constant used post-Phase 18 (e.g. `PROFILE_DIMENSIONS_COLUMN`)
- Apply the 3.0 dimension weights exactly as defined in `Compatibility Scoring And Matching Algorithm 3.0`
- Incomplete dimension coverage:
  - include profile in scoring
  - apply deterministic confidence attenuation (do not exclude)
- Remove any 2.0 factor names from scoring logic and tests
- Update `scripts/run-matching-job.mjs` call sites to use the updated scoring function/shape

## Deliverables

- Updated scoring functions in `packages/core/`
- Updated scoring call sites in `scripts/run-matching-job.mjs`
- Unit tests:
  - aligned dimension pair scores higher than mismatched dimension pair
  - incomplete (3/6) profile scores lower than complete (6/6) given otherwise similar inputs
- Grep proof: no 2.0 factor names remain in scoring code

## Verification Checklist

- `pnpm lint`
- `pnpm typecheck`
- `pnpm test`
- `pnpm build`

## Completion Criteria

- Matching scoring uses only the 3.0 six-dimension model + weights
- Tests demonstrate relative scoring + incomplete handling
- No legacy factor-name references remain
