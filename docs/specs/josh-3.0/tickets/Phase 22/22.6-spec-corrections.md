## File 2: Profile_Interview_And_Signal_Extraction_Spec_3_0.md — Full Rewrite

This spec currently documents the 2.0 twelve-factor Friend Fingerprint system. It must be replaced with documentation of the 3.0 six-dimension coordination model and holistic extraction process.

**File location:** `docs/specs/Profile_Interview_And_Signal_Extraction_Spec_3_0.md` (confirm exact path in the codebase — the filename may differ slightly from this).

### Content To Remove

Every reference to the following must be removed from the rewritten spec:

- The twelve-factor Friend Fingerprint and any of the 2.0 factor names
- `InterviewExtractInput` and `InterviewExtractOutput` interfaces with `stepId`
- `fingerprintPatches` or `FINGERPRINT_FACTOR_KEYS`
- Step-by-step extraction (where each interview step targets a specific fingerprint factor)

### Content To Document

The rewritten spec must cover:

**1. The 6-Dimension Coordination Profile (Layer A)**

Document each dimension with its name, description, data type, and what it captures:

- `social_energy` — preference for high-stimulation vs. low-stimulation social environments; range_value 0–1, confidence 0–1, freshness_days int
- `social_pace` — preference for spontaneous vs. structured plans; same shape
- `conversation_depth` — preference for surface-level vs. substantive conversation; same shape
- `adventure_orientation` — preference for familiar vs. novel experiences; same shape
- `group_dynamic` — preference for leading vs. following vs. collaborating; same shape
- `values_proximity` — importance of shared values and worldview in connections; same shape

Each dimension has `range_value` (float 0–1), `confidence` (float 0–1), `freshness_days` (integer). Document this structure explicitly.

**2. The 3 Coordination Signals**

- `scheduling_availability` — time buckets when the user is typically available
- `notice_preference` — how far in advance plans need to be confirmed
- `coordination_style` — how the user prefers to handle logistics

**3. HolisticExtractInput and HolisticExtractOutput Interfaces**

Document the schemas used by the LLM holistic extraction call. Base these on whatever interfaces exist in `packages/llm/` or `packages/core/` after Phase 18 — open those files to get the exact shape before writing the spec. The spec must reflect the actual code, not an approximation.

HolisticExtractInput shape (confirm against code):

```typescript
interface HolisticExtractInput {
  conversationHistory: ConversationTurn[];
  conversationFocus: string; // which dimension is the current interview targeting
  currentProfile: Partial<CoordinationDimensions>;
}
```

HolisticExtractOutput shape (confirm against code):

```typescript
interface HolisticExtractOutput {
  dimensions: Partial<CoordinationDimensions>;
  coordinationSignals: Partial<CoordinationSignals>;
  activityPatterns: ActivityPattern[];
  coverageSummary: Record<string, number>; // dimension -> current confidence
  needsFollowUp: string[];
}
```

**Important:** Open the actual source files in `packages/llm/` and `packages/core/src/profile/` to get the exact interface shapes. Write the spec to match the code as it exists after Phase 18. If the interfaces differ from the above sketch, the spec must reflect the actual interfaces.

**4. Holistic Extraction Trigger Points**

Document when the holistic extraction LLM call fires:

- After every user reply during the interview
- At the abbreviated interview wrap for invited users
- When the signal coverage tracker detects a gap that needs targeted follow-up

**5. Completeness Thresholds**

Document both threshold sets:

complete_mvp requirements:

- All 6 coordination dimensions at confidence >= 0.55
- All 3 coordination signals captured at confidence >= 0.60
- At least 3 activity patterns at confidence >= 0.60
- group_size_pref captured
- time_preferences captured (at least one bucket)

complete_invited requirements:

- At least 3 coordination dimensions at confidence >= 0.45
- scheduling_availability signal captured
- At least 1 activity pattern at confidence >= 0.50

**6. Confidence Update Rules**

Document the constraints on LLM extraction outputs (from the LLM System Prompt Guidance section of the Conversation Behavior Spec):

- No dimension's range_value may move more than 0.25 in a single extraction pass
- Confidence may not exceed 0.75 in a single extraction pass
- Inferred signals must have confidence 0.10–0.15 lower than directly extracted signals
- Fields must be absent rather than guessed when evidence is weak
