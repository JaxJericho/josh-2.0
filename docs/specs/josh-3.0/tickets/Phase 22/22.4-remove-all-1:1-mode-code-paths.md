# Ticket 22.4 — Remove All 1:1 Mode Code Paths

## Phase And Context

Phase 22 cleanup. This ticket removes all remaining one_to_one references from application code. Earlier work in Phase 21 removed one_to_one from the matching job (Ticket 21.2) and from the schema (Ticket 21.5). This ticket confirms no stray references remain anywhere else in the codebase.

Ticket 21.2 and 21.5 must be fully complete before this ticket runs.

---

## Pre-Work: Required Grepping

Run all of the following before writing any code.

    grep -rn "one_to_one" --include="*.ts" --include="*.mjs" packages/ apps/ scripts/ lib/
    grep -rn "OneToOne" --include="*.ts" packages/ apps/ scripts/ lib/
    grep -rn "match_mode" --include="*.ts" packages/
    grep -rn "z.enum" --include="*.ts" packages/ | grep -i "one_to_one"

For every result returned, determine whether it is a migration file (leave it), a runtime file (remove or update), or a documentation/runbook file (update in place). Capture the full output before making any edits.

---

## Likely Locations To Check

Even if the grep above returns nothing for these, verify each explicitly:

- packages/core/src/types/ - any MatchMode type or MATCH_MODES constant
- packages/db/src/types/ - any type that includes one_to_one as a valid value
- Any Zod schema that validates match_runs.mode or match_candidates.mode
- Any utility that creates match_runs rows and sets mode
- Any test that creates a match_runs row with mode: "one_to_one"
- Dashboard or admin code that displays match mode labels

---

## What To Do For Each Reference

TypeScript union type:

    // Before
    type MatchMode = "linkup" | "one_to_one";
    // After
    type MatchMode = "linkup";

Zod validator:

    // Before
    z.enum(["linkup", "one_to_one"])
    // After
    z.enum(["linkup"])

Runtime constant (if still present - should have been removed in 21.2):
Remove one_to_one from any Set or array of valid modes.

Tests:

- Remove tests that pass mode: "one_to_one" and expect success
- Keep tests that confirm --mode one_to_one throws (valid regression guards from Ticket 21.2)

Documentation and runbooks:
Update in place. Document that 1:1 match mode was removed in JOSH 3.0 Phase 21, not silently deleted.
