# Ticket 2.2 â€” Remove Retired Intent Types and Session Modes (M)

## Context

This is the second ticket in Session 2. It runs after Ticket 2.1. With the three user-initiated handlers deleted, the intent types and session modes they depended on are now dead code in the TypeScript layer. This ticket removes them from all application-layer type definitions, Zod schemas, classifier logic, and LLM prompts.

**Critical scope boundary:** `awaiting_social_choice` and `pending_plan_confirmation` exist in the Postgres `conversation_mode` enum (added in Phase 16.1). Dropping DB enum values requires the destructive enum recreation pattern (create new enum, alter column, drop old). That work is deferred to a future cleanup session. This ticket removes them from TypeScript only.

Read `AGENTS.md` and `Claude_Code_Instructions.md` before writing any code.

---

## Goal

Remove `OPEN_INTENT`, `NAMED_PLAN_REQUEST`, and `PLAN_SOCIAL_CHOICE` from the TypeScript intent type system. Remove `awaiting_social_choice` and `pending_plan_confirmation` from the TypeScript `ConversationMode` union. Update the intent classifier and LLM prompt to reflect the pivot model's intent taxonomy. Do not touch the DB schema.

```ts

  | "INTERVIEW_ANSWER"
  | "INTERVIEW_ANSWER_ABBREVIATED"
  | "INVITATION_RESPONSE" // new in pivot model (handled by handleInvitationResponse, Phase 3)
  | "CONTACT_INVITE_RESPONSE" // member-to-member referral path, retained
```

**Note on `INVITATION_RESPONSE`:** The handler `handleInvitationResponse` is implemented in Phase 3 (Ticket 3.1). Add this intent type now so the type system is consistent with the pivot model from this point forward. The handler will be wired into the router in Phase 3.
