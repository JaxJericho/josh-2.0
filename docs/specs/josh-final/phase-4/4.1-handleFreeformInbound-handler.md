LLM Classification Prompt
Define as a versioned constant in packages/llm/src/prompts/. Bump PROMPT_VERSION on every change.
System: You are classifying an inbound SMS message from a JOSH user who is not in any active flow.
Classify into exactly one category:

- AVAILABILITY_SIGNAL: user expresses they are free, available, or looking for something to do (examples: "I'm free Saturday", "nothing going on this weekend", "bored")
- POST_EVENT_SIGNAL: user references an activity they recently attended or comments on a past event (examples: "had a great time last night", "that coffee place was perfect")
- PREFERENCE_UPDATE: user expresses a preference, boundary, or category opt-out/in (examples: "stop sending me hiking stuff", "I actually love early mornings", "not into group things right now")
- GENERAL_FREEFORM: anything else

Return valid JSON only, no markdown:
{ "category": "<CATEGORY>", "summary": "<one sentence summary of the message>" }
Timeout: 5 seconds. Single retry on network error. On timeout or LLM failure: treat as GENERAL_FREEFORM — do not throw.

Implementation
File: packages/messaging/src/handlers/handle-freeform-inbound.ts
Routing per category
AVAILABILITY_SIGNAL
Write to learning_signals:
{ user_id: context.userId, signal_type: 'availability_expressed', payload: { summary: llmResult.summary, raw_message: context.messageText }, correlation_id: context.correlationId,}
Respond: "Good to know — JOSH will factor that in. Look out for something soon."
Set mode to idle.
POST_EVENT_SIGNAL
Query: does this user have an accepted invitations row with responded_at > now() - interval '7 days'?
If yes: call handlePostActivityCheckin(context) with the synthesized context. Let that handler manage the session.
If no: treat as GENERAL_FREEFORM — there is no matching recent invitation to anchor the check-in.
PREFERENCE_UPDATE
Call the preference extraction LLM with a focused prompt (reuse the holistic extractor call pattern — target the preference/boundary signals only, not the full 6-dimension extraction). Timeout: 5 seconds. On failure: respond with "Noted — JOSH will keep that in mind." and skip the profile write.
On success: write the preference update to profiles via profile-writer.ts. Write a profile_events row explaining the change.
Respond: "Noted — JOSH will keep that in mind."
Set mode to idle.
GENERAL_FREEFORM
Respond: "JOSH handles plans and invitations over text — no app needed. JOSH will be in touch with something tailored to you. Reply HELP for options."
Set mode to idle.
