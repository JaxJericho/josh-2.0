## Implementation

### File: `packages/core/src/invitation/dispatch-invitation.ts`

```ts
export type DispatchParams = {
  userId: string;
  invitationType: "solo" | "linkup";
  activityKey: string;
  proposedTimeWindow: string;
  locationHint?: string;
  linkupId?: string; // required when invitationType = 'linkup'
  correlationId: string;
};

export type DispatchResult =
  | { dispatched: true; invitationId: string }
  | {
      dispatched: false;
      reason:
        | "eligibility_gate"
        | "cooldown"
        | "weekly_cap"
        | "backoff_suppressed"
        | "already_invited_this_week"
        | "quiet_hours";
    };

export async function dispatchInvitation(
  params: DispatchParams,
): Promise<DispatchResult>;
```

#### Steps

**1. Eligibility check**

Call `checkInvitationEligibility(userId)`. If not eligible, return `{ dispatched: false, reason }` immediately ‚Äî no side effects.

**2. Idempotency check**

Compute `idempotencyKey = INVITATION_IDEMPOTENCY_KEY(userId, activityKey, isoWeek(now()))`.

Query `invitations` where `idempotency_key = idempotencyKey`. If a row exists: return `{ dispatched: false, reason: 'already_invited_this_week' }`.

**3. Quiet hours check**

Determine the user's region timezone (query `users JOIN regions` ‚Äî use `America/Los_Angeles` if `regions.timezone` is not present). Convert `now()` to local time. If local hour is 22‚Äì23 or 0‚Äì7 (i.e., 22:00‚Äì07:59 local time): return `{ dispatched: false, reason: 'quiet_hours' }`.

**4. Snapshot group_size_preference**

Load `profiles.group_size_preference` for the user. This value is snapshotted into the `invitations` row at dispatch time. If null, store null (quorum evaluator defaults to `{ min: 2, max: 10 }`).

**5. Transaction**

Inside a single transaction:

- Insert `invitations` row:
  ```ts
  {
    user_id: userId,
    invitation_type: invitationType,
    activity_key: activityKey,
    proposed_time_window: proposedTimeWindow,
    location_hint: locationHint ?? null,
    linkup_id: linkupId ?? null,
    state: 'pending',
    offered_at: now(),
    expires_at: now() + INVITATION_EXPIRY_HOURS hours,
    group_size_preference_snapshot: groupSizePreference,
    idempotency_key: idempotencyKey,
    correlation_id: correlationId,
  }
  ```
- Set `conversation_sessions.mode = 'awaiting_invitation_response'`
- Update `users.last_invited_at = now()`
- Update weekly counter:
  - If `invitation_week_start` is null or stale (> 7 days ago): set `invitation_week_start = date_trunc('week', now())`, `invitation_count_this_week = 1`
  - Otherwise: increment `invitation_count_this_week`

**6. Enqueue outbound SMS**

Write to `sms_outbound_jobs` (or the existing outbound queue pattern). Use the SMS copy below. Do this inside the transaction if the outbound queue is a DB table; outside if it is a separate service.

**7. Return**

Return `{ dispatched: true, invitationId: invitation.id }`.

---

#### SMS Copy

Solo:

> "Hey [first_name] ‚Äî JOSH found something for you: [activity_display_name] [proposed_time_window]. Interested? Reply YES to confirm or PASS to skip."

Linkup:

> "Hey [first_name] ‚Äî JOSH found a group activity that fits you: [activity_display_name] [proposed_time_window] with a small group. Reply YES to join or PASS to skip."

Resolve `activity_display_name` from `activity_catalog` via `activity_key`. Fall back to title-cased `activity_key` if lookup fails.
