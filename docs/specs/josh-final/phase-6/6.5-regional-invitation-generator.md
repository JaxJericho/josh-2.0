## Implementation

### File: `packages/core/src/invitation/regional-invitation-generator.ts`

```ts
export type GeneratorRunResult = {
  regionId: string;
  eligibleUserCount: number;
  clustersFormed: number;
  clusterMembersDispatched: number;
  soloInvitationsDispatched: number;
  groupInvitationsDispatched: number;
  skippedQuietHours: number;
  skippedActivityUnresolvable: number;
  skippedTimeWindowUnresolvable: number;
  errors: string[];
  durationMs: number;
};

export async function runRegionalGenerator(
  regionId: string,
): Promise<GeneratorRunResult>;
```

#### Steps

**1. Advisory lock**

```sql
SELECT pg_try_advisory_lock(hashtext($1)) as acquired
```

Where `$1` is the `regionId`. If `acquired = false`: log `{ event: 'generator_run.lock_unavailable', regionId }` and return an empty result with all counts at 0. Do not throw.

**2. Load eligible users**

```sql
SELECT
  u.id as user_id,
  u.last_invited_at,
  u.invitation_count_this_week,
  u.invitation_week_start,
  u.invitation_backoff_count,
  p.coordination_dimensions,
  p.activity_patterns,
  p.scheduling_availability,
  p.boundaries,
  p.interest_signatures,
  p.relational_context,
  p.group_size_preference
FROM users u
JOIN profiles p ON p.user_id = u.id
JOIN subscriptions s ON s.user_id = u.id AND s.status = 'active'
WHERE u.region_id = $regionId
  AND u.state = 'active'
  AND p.profile_state IN ('complete_mvp', 'complete_full')
  -- complete_invited hard filter: never relax
  AND p.profile_state != 'complete_invited'
  AND u.safety_hold IS NULL
```

Then filter in application code: call `checkInvitationEligibility(userId)` for each user and retain only those returning `{ eligible: true }`. (The eligibility check is in application code, not SQL, because it requires business logic beyond simple column checks.)

Build `EligibleUser[]` from the results, normalizing each profile via the profile normalizer. Default `group_size_preference` to `{ min: 2, max: 10 }` if null.

**3. Detect clusters**

Call `detectClusters(eligibleUsers)`. Receives `Cluster[]` — any user not in a cluster is unclustered.

Compute unclustered users:

```ts
const clusteredUserIds = new Set(clusters.flatMap((c) => c.members));
const unclusteredUsers = eligibleUsers.filter(
  (u) => !clusteredUserIds.has(u.userId),
);
```

**4. Create system LinkUps for clusters**

For each cluster where `activityKey` is not null (not `activity_unresolvable`) and `proposedTimeWindow` is set:

- Insert a `linkups` row:
  ```ts
  {
    system_created: true,
    initiator_user_id: null,
    state: 'broadcasting',
    activity_key: cluster.activityKey,
    proposed_time_window: cluster.proposedTimeWindow,
    min_size: 2,
    max_size: MAX_CLUSTER_SIZE,
    region_id: regionId,
    correlation_id: correlationId,
  }
  ```
- For each cluster member, call `dispatchInvitation({ userId, invitationType: 'linkup', activityKey, proposedTimeWindow, linkupId, correlationId })`
- Collect results: track dispatched, skippedQuietHours, errors

For `activity_unresolvable` or time-window-unresolvable clusters: add all members to `unclusteredUsers` list.

**5. Solo fallback**

For each unclustered user:

- Call `selectSoloInvitation(userId)`
- If candidate returned: call `dispatchInvitation({ userId, invitationType: 'solo', activityKey, proposedTimeWindow, locationHint, correlationId })`
- If null returned (no eligible activity): log and skip

**6. Re-queue quiet hours skips**

For each dispatch that returned `{ dispatched: false, reason: 'quiet_hours' }`:

- Enqueue a QStash job targeting `dispatchSingleUserInvitation(userId)` at `08:00 local time + randomInt(0, 30) minutes` the next day
- Use QStash `delay` parameter

**7. Release advisory lock**

```sql
SELECT pg_advisory_unlock(hashtext($regionId))
```

Always release in a `finally` block — lock must be released even if the run throws.

**8. Log summary**

Write to `message_events` (or equivalent structured log table):

```ts
{
  event_type: 'generator_run_complete',
  payload: generatorRunResult,
  correlation_id: correlationId,
  created_at: now(),
}
```

**9. Return** `GeneratorRunResult`.

## Hard Rules for This Ticket

- The `complete_invited` hard filter in the SQL query is mandatory. Comment: `// complete_invited hard filter: never relax`. This applies even if the eligible pool is empty.
- Advisory lock must be released in a `finally` block — no path through the function releases it conditionally
- Errors on individual user dispatch do not abort the run — log and continue
- The SQL advisory lock is the only acceptable raw SQL in this file — document the exception
- `system_created = true` and `initiator_user_id = null` must be set on every system-created LinkUp row — the application invariant from Ticket 1.3
